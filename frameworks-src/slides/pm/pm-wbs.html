WBS — Work Breakdown Structure
pm-01
WBS とは
プロジェクトの成果物・作業を階層的に分解し、管理可能なワークパッケージに落とし込む。スコープ可視化・工数見積もり・進捗管理の基盤。
WBS 作成のポイント
  • 成果物（名詞）で分解する
  • 最小単位は 1〜2 週間以内の作業
  • 100%ルール：上位 = 下位の総和
  • 「誰が・何を・いつまでに」を明確に
WBS 階層例（Web システム開発）
▶ Web システム開発
┣ 要件定義
ヒアリング / 要件書作成 / レビュー・承認
┣ 設計
基本設計 / 詳細設計 / DB 設計 / API 設計
┣ 開発・実装
┣ テスト
単体 / 結合 / システム / UAT
┗ リリース・移行
