Facade — シンプルな窓口
gof-10
意図
複雑なサブシステムに対してシンプルなインタフェース（窓口）を提供する。クライアントをサブシステムの複雑さから隔離する。
使いどころ
  • 複数の複雑なライブラリをまとめて使う場合
  • レイヤードアーキテクチャのサービス層
  • サードパーティAPIのラッパー
クラス図
Client → Facade
　　　　　　↓
　　SubsystemA SubsystemB SubsystemC
Java 実装例（動画変換）
// 複雑なサブシステム群
class VideoDecoder {
    VideoFile decode(String path) { /*...*/ return new VideoFile(); }
}
class AudioMixer { void mix(VideoFile v) { /*...*/ } }
class Encoder {
    void encode(VideoFile v, String fmt) { /*...*/ }
}
class BitrateReader {
    void adjust(VideoFile v) { /*...*/ }
}

// Facade — 複雑な操作を1メソッドに隠蔽
class VideoConverter {
    private VideoDecoder decoder = new VideoDecoder();
    private AudioMixer   mixer   = new AudioMixer();
    private Encoder      encoder = new Encoder();
    private BitrateReader reader = new BitrateReader();

    public void convert(String inputPath, String format) {
        VideoFile video = decoder.decode(inputPath);
        mixer.mix(video);
        reader.adjust(video);
        encoder.encode(video, format);
        System.out.println("変換完了: " + format);
    }
}

// 使用例 — クライアントはFacadeだけ知ればよい
VideoConverter converter = new VideoConverter();
converter.convert("movie.mp4", "avi");
