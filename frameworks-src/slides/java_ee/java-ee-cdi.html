CDI（Contexts and Dependency Injection）
オブジェクトの生成・ライフサイクル・注入をコンテナに委ねる。Spring の @Autowired に相当するのが @Inject。
@ApplicationScoped          // アプリで 1 インスタンス
public class UserService {
    @Inject
    UserRepository repo;    // コンテナが自動注入

    public User find(Long id) {
        return repo.findById(id);
    }
}

// CDI スコープ一覧
@RequestScoped    // 1 リクエストで 1 インスタンス
@SessionScoped    // セッションで 1 インスタンス
@ApplicationScoped // アプリ全体で 1 インスタンス
@Dependent        // 注入先と同じスコープ（デフォルト）
EJB（Enterprise JavaBeans）
トランザクション管理・セキュリティ・リモート呼び出し等をアノテーションで宣言できる。現在は CDI + JTA + Security で代替できるケースが多い。
種別特徴
@Stateless状態を持たない。最も一般的。スレッドセーフ
@Statefulクライアントセッションごとに状態を保持
@Singletonアプリで 1 インスタンス。@Lock で並行制御
@MessageDrivenJMS メッセージを非同期受信
@Stateless
@TransactionAttribute(REQUIRED)  // トランザクション自動管理
public class OrderService {
    @Inject UserService userService;

    public void placeOrder(Order order) {
        // コンテナがトランザクションを開始・コミット
        validate(order);
        userService.deductPoints(order.userId());
        persist(order);
    }
}
