JPA の基本概念
用語説明
EntityDB テーブルに対応するクラス（@Entity）
EntityManagerEntity の CRUD・クエリを行うコア API
PersistenceContextEntityManager が管理する Entity のキャッシュ（1 次キャッシュ）
JPQLテーブルではなく Entity クラスを対象とするクエリ言語
Lazy / Eager関連 Entity のロード戦略（FetchType）
Entity 実装例
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(
        strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String name;

    @OneToMany(mappedBy = "user",
               fetch = FetchType.LAZY,
               cascade = CascadeType.ALL)
    private List orders;
}
EntityManager の操作
@PersistenceContext
EntityManager em;

// 作成
User user = new User("Alice");
em.persist(user);

// 主キーで取得（1 次キャッシュを利用）
User found = em.find(User.class, 1L);

// JPQL クエリ
List list = em.createQuery(
    "SELECT u FROM User u"
    + " WHERE u.name = :name", User.class)
    .setParameter("name", "Alice")
    .getResultList();

// 削除
em.remove(found);
N+1 問題と JOIN FETCH
Lazy ロードの関連を forEach でアクセスすると N+1 本の SQL が発行される。
対策：SELECT u FROM User u JOIN FETCH u.orders で一括取得する。
