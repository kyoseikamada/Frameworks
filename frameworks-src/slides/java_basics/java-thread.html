スレッド生成・実行の方法
// ① Runnable（基本）
Thread t = new Thread(() ->
    System.out.println("running"));
t.start();

// ② ExecutorService（実務標準）
ExecutorService pool =
    Executors.newFixedThreadPool(4);
Future f = pool.submit(() -> fetch());
String result = f.get(); // ブロッキング
pool.shutdown();

// ③ CompletableFuture（非同期チェーン）
CompletableFuture
    .supplyAsync(() -> fetchData())
    .thenApply(data -> process(data))
    .thenAccept(r -> save(r))
    .exceptionally(ex -> { log(ex); return null; });

// ④ Virtual Thread（Java 21+）
try (var exec =
        Executors.newVirtualThreadPerTaskExecutor()) {
    exec.submit(() -> handleRequest());
}

手段特徴・用途
synchronizedメソッド・ブロック単位のモニターロック
volatile変数の可視性保証。read/write のアトミック性
ReentrantLocktryLock・タイムアウトなど細かい制御
AtomicInteger 等CAS によるロックフリー操作
ConcurrentHashMapセグメントロックで高スループット
BlockingQueueスレッド間のプロデューサ・コンシューマ
Virtual Thread（Project Loom）
Java 21 正式リリース。JVM が管理する超軽量スレッド。I/O 待ちでも OS スレッドをブロックしない。
従来の非同期コールバック地獄なしに、1 リクエスト 1 スレッドの同期スタイルで高スループットを達成できる。
