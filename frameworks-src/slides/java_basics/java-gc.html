Heap の世代構造
┌────────────────────────────────┐
│          Young 世代              │ ← Minor GC（頻繁・短時間）
│  Eden │ Survivor0 │ Survivor1  │
├────────────────────────────────┤
│          Old 世代                │ ← Major GC（低頻度・長時間）
└────────────────────────────────┘

新規オブジェクト → Eden に割り当て
Minor GC 生き残り → Survivor へ
一定回数生存 → Old 世代へ昇格
GC の種類
GC特徴主な用途
Serialシングルスレッド小規模・組込み
Parallel高スループットバッチ処理
G1 GC停止時間予測可能。Java 9 デフォルト汎用
ZGC停止 <1ms。大ヒープ対応低レイテンシ
ShenandoahZGC 同等の低レイテンシ低レイテンシ
GC チューニングの基本
# G1GC 推奨設定
-XX:+UseG1GC
-Xms4g -Xmx4g             # 初期=最大で固定
-XX:MaxGCPauseMillis=200   # 目標停止時間(ms)

# GC ログ（Java 11+）
-Xlog:gc*:file=gc.log:time,uptime

# OOM 時に Heap ダンプ
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/tmp/heap.hprof
チューニングの原則
  • まず計測：GC ログを取得してから判断する
  • Heap を大きくしすぎない：Major GC の停止時間が増える
  • Young 世代を適切に：短命オブジェクトは Young で回収させる
  • オブジェクト生成を減らす：そもそも GC の根本対策
  • 初期=最大サイズを揃える：Heap リサイズのオーバーヘッドを防ぐ
