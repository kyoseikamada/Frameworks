Java メモリモデル（JMM）
マルチスレッドでのメモリの可視性と操作順序を規定する仕様。各スレッドは CPU キャッシュに値を持つため、明示的な同期なしに他スレッドの変更は見えない。
概念内容
happens-beforeA happens-before B なら、A の Write が B から可視
volatilevolatile Write → 後続の volatile Read に happens-before
synchronizedロック解放 → 次の同じロック取得に happens-before
Thread.start()start() 前のコード → スレッド内コードに happens-before

例外の階層構造
Throwable
├── Error  ← 通常 catch しない
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── AssertionError
└── Exception
    ├── RuntimeException  ← 非チェック例外
    │   ├── NullPointerException
    │   ├── IllegalArgumentException
    │   └── IndexOutOfBoundsException
    └── IOException  ← チェック例外
        ├── FileNotFoundException
        └── SocketException
チェック vs 非チェック：チェック例外はコンパイラが catch / throws を強制する。実務では Spring 等の影響もあり RuntimeException 系が主流。ライブラリ境界では検査例外も有効。
