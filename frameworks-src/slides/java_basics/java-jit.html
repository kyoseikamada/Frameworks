JIT（Just-In-Time）コンパイルとは
実行時に頻繁に呼ばれるバイトコードをネイティブコードに変換してキャッシュする。2 回目以降は高速なネイティブコードが実行される。
HotSpot JIT の段階
1
インタプリタ実行：起動直後。バイトコードを 1 命令ずつ解釈実行。プロファイル情報を収集
2
C1 コンパイル（クライアントコンパイラ）：軽い最適化。呼び出し回数の閾値で起動
3
C2 コンパイル（サーバーコンパイラ）：プロファイル情報をもとにアグレッシブな最適化
JIT が行う主な最適化
最適化内容
インライン展開小さなメソッド呼び出しを呼び出し元に埋め込み、呼び出しオーバーヘッドを削減
エスケープ解析Heap ではなく Stack に割り当て可能なオブジェクトを判定して GC 負荷を削減
ループ展開ループの条件判定・インクリメントのオーバーヘッドを削減
投機的最適化実行時の型情報をもとに最速コードを生成。外れたら Deoptimize して再コンパイル
ウォームアップ問題と GraalVM
JIT が効果を発揮するまで時間がかかる（ウォームアップ）。起動直後のレイテンシが高くなる問題がある。

GraalVM Native Image：AOT（Ahead-of-Time）コンパイルでネイティブバイナリを生成。起動時間をミリ秒単位に短縮。Quarkus / Micronaut が活用。
