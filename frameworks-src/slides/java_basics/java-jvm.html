Java 実行の全体像
ソースコード (.java)
      ↓  javac（コンパイラ）
バイトコード (.class)
      ↓  java（JVM起動）
  ┌──────────────────────────────┐
  │        JVM                    │
  │  ┌─────────────┐             │
  │  │ Class Loader │ .classを読込│
  │  └─────────────┘             │
  │  ┌────────────────────────┐  │
  │  │ Runtime Data Areas     │  │
  │  │  Heap / Stack / Method │  │
  │  └────────────────────────┘  │
  │  ┌──────────────────────┐    │
  │  │ Execution Engine      │    │
  │  │  JIT Compiler         │    │
  │  │  Interpreter          │    │
  │  └──────────────────────┘    │
  └──────────────────────────────┘
         ↕  JNI
    ネイティブコード（OS依存）
JVM のメモリ領域
領域内容共有
Heapオブジェクト・配列。GC対象。Young / Old 世代に分かれる全スレッド
Metaspaceクラス定義・定数プール。Java 8 で PermGen を置換全スレッド
Stackメソッド呼び出しフレーム・ローカル変数スレッド専有
PC Register現在実行中の命令アドレススレッド専有
Native StackJNI 呼び出し用スレッド専有
主要な JVM 起動オプション
-Xms512m              # Heap 初期サイズ
-Xmx2g                # Heap 最大サイズ
-XX:+UseG1GC          # G1GC を使用
-XX:+PrintGCDetails   # GC ログ出力
-verbose:class        # クラスロード状況
-XX:+HeapDumpOnOutOfMemoryError
「Write Once, Run Anywhere」の実現手段。.class ファイルはどの OS の JVM でも動く。
