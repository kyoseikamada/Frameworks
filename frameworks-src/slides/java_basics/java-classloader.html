クラスローダーの階層
Bootstrap ClassLoader
  ↑ 親委譲
Platform ClassLoader（Java 9+）
  ↑ 親委譲
Application ClassLoader
  （-classpath 上のクラス）
  ↑ 親委譲
User-defined ClassLoader
  （独自実装・Tomcat 等）
親委譲モデル（Parent Delegation）
クラスのロード要求を受けると、まず親に委譲する。見つからなければ自分でロードする。java.lang.String など JDK のコアクラスが上書きされるのを防ぐセキュリティ機構。
カスタムクラスローダーの用途
  • Tomcat / OSGi：Web アプリごとにクラス空間を分離
  • 暗号化された .class ファイルを復号してロード
  • プラグイン機構（IDE プラグイン等）の実現
クラスのロード → リンク → 初期化
フェーズ内容
Loading.class をバイト列として読み込む
Verificationバイトコードが仕様に適合しているか検証
Preparationstatic 変数にデフォルト値を割り当て
Resolutionシンボリック参照を実体参照に変換
Initializationstatic 初期化子・static 変数の初期化を実行
